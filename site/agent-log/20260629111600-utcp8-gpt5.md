### 2. 根因诊断

抓 dev server 渲染出的 HTML head，对比 `Layout.astro` 源码：

```html
<!-- Layout.astro 源码 -->
<script src="https://cdn.tailwindcss.com?plugins=forms,container-queries"></script>

<!-- 实际渲染输出 -->
<script type="module" src="/src/layouts/Layout.astro?astro&type=script&index=0&lang.ts"></script>
```

**根因**：Astro 5 默认行为是**把所有 `<script>` 当 module 处理并强制 `type="module"`**。即便 src 是外部 HTTPS 链接，Astro 也会重写成本地 module 路径，让浏览器去 import 这个 URL，CDN 脚本因此永远不会被执行。

### 3. 修复

在 Layout.astro 的 Tailwind CDN script 标签上加 `is:inline`：

```diff
- <!-- Tailwind CDN -->
- <script src="https://cdn.tailwindcss.com?plugins=forms,container-queries"></script>
+ <!-- Tailwind CDN: 必须 is:inline，否则 Astro 5 会把外部 src 改写成本地 module，CDN 不会执行 -->
+ <script is:inline src="https://cdn.tailwindcss.com?plugins=forms,container-queries"></script>
```

`is:inline` 告诉 Astro 不要处理这个 script，原样输出。
