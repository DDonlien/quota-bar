## 用户原始 prompt

几个问题：打开以后，如果点击了刷新但还没有给权限，还是会错误的显示2个占位符（claude和minimax，codex是对的），点击刷新不应该把这个dropdown页面关掉，应该直接刷新出来；收费的金额应该按照实际订阅的地区，结合使用者的语言来判断它的货币，并转换成对应汇率。

比如说 ChatGPT 的 Codex Plus 是 $20，我用简体中文，你就应该以汇率换算成人民币。

## 行动记录

- 执行 `git pull`，结果为 `Already up to date`。
- 修复权限未授权时的占位展示：`RefreshCoordinator` 先从所有 snapshot 里计算 `needsFullDiskAccess`，再只展示 `available` 且带真实 quota 的 snapshot。这样 Claude/MiniMax 这类权限/未接入 fallback 不再出现在 dropdown。
- 修复“立即刷新”关闭 dropdown：将原生 `NSMenuItem` 替换为承载 SwiftUI Button 的 custom view，点击后调用 `coordinator.refreshNow()`，不触发 NSMenu 原生关闭行为。
- 修复价格本地化：
  - 新增 `PreferredCurrency`，按系统地区/首选语言选择展示币种；简体中文或中国区默认 CNY。
  - 新增 `ExchangeRateProvider`，通过 `https://open.er-api.com/v6/latest/USD` 获取 USD→CNY 汇率，缓存 12 小时，3 秒超时，失败回退本地保守汇率。
  - `ProviderPricing.localizedMonthlyPrice` 将官方 USD 基准价格转换为本地货币字符串。
  - `DashboardState.totalMonthlyCostText` 使用当前展示币种汇总，不再把 `US$20` 解析成 `¥20`。
- 更新 `README.md` 和 `REQUIREMENTS.md`，记录刷新不关闭菜单与价格本地化规则。
