# 对话与行动记录

1. **需求解读**：用户希望在 dropdown 中「每个大供应商价格左边」展示一个灰色日期标签，语义是「数据/订阅最后有效日」（`2026/6/25` → 6/25 仍有效，26 起过期）。
2. **数据可用性**：
   - `QuotaWindow.resetsAt` 已有，最自然代理就是「该 provider 所有 quota 中最晚的 resetsAt」（`max(quotas.map(\.resetsAt))`）。
   - 现有 dashboard parsers 都没有直接返回「订阅账单日 / next bill date」字段；如果将来某 provider 解析出更精确的值，可通过 `subscriptionExpiresAt` 显式参数覆盖默认推断。
3. **方案**：
   - 在 `ProviderSnapshot` 新增 `subscriptionExpiresAt: Date?`；`init` 在没显式传入时默认取 `max(quotas.compactMap(\.resetsAt))`。
   - `.loading` 工厂：保持 `nil`（没 quota → 没日期）。
   - 现有 15 处 `ProviderSnapshot(...)` 调用点全部走默认推断，**不需要逐一改**。
   - `MenuView.PlanHeader`：当 `monthlyPrice != nil` 且 `availability == .available` 且 `subscriptionExpiresAt != nil` 时，在价格左侧展示 `Text(expiresAtText)`（`yyyy/M/d`、11pt、secondary 灰、等宽数字），与价格间距 6pt；`needsConfiguration` 分支保持原样（隐藏按钮，无日期）。
4. **worktree 处理**：
   - `git worktree add -b feat/price-update-date .worktrees/price-update-date main` 建立新 worktree。
   - 第一次 `swift build` 因为 `.build` 未带过来 + 网络 fetch 失败，复制 main worktree 的 `.build` + `Package.resolved`，并清掉 stale module cache（绝对路径指向原 worktree），再 build 通过。
5. **改动**：
   - `QuotaModels.swift`：`ProviderSnapshot` 加 `subscriptionExpiresAt` 字段 + init 默认推断 + `.loading` 工厂传 `nil`。
   - `MenuView.swift`：`MenuDashboardStyle` 新增 `planExpiresAtFontSize = 11` / `planPriceTrailingGap = 6`；`PlanHeader` 新增 `expiresAtText` 计算属性 + `DateFormatter` 静态实例；右侧 HStack 包住「日期 + 价格」并在 needsConfiguration 之外渲染。
   - `DESIGN.md`：字体段加「计划头部最后有效日期」条目；组件风格段更新「服务标题行」描述。
   - `REQUIREMENTS.md`：v0.3.0 UI-A 块加 `[0.3.0-UI-A-010]`，已勾选完成。

# 备注

- **未提交**：保持 worktree 工作区原样，方便用户先看 diff 再决定是否合并到 main。
- **不在 main 留下改动**：所有改动都落在 worktree（`.worktrees/price-update-date/`），main worktree 的 dirty 状态（v0.3.0 收尾）原封未动。
- **默认推断的语义**：`max(quota.resetsAt)` 是「当前显示的剩余额度过期最晚的那条」，与用户描述的「什么时候过期」在直觉上吻合。
  - Codex 5h + 周额度 → 取周额度的 resetsAt（≈ 7 天后）。
  - Kimi Code + Work → 取 Work 月额度的 resetsAt。
  - MiniMax / Antigravity 多订阅组：每个订阅组各自的 `max(resetsAt)` 仍按 subscriptionGroup 隔离。
- **后续可扩展点**：
  - 某 provider parser 如果能从 API 拿到更明确的「订阅账单日 / 服务到期日」（不是 quota reset），可以在 `fetchSnapshot` 末尾显式传 `subscriptionExpiresAt:` 覆盖默认推断。
  - 当前没有「数据有效性」独立的字段；如果将来想做「数据陈旧提醒」（`isStale` 之外），可以再加 `dataStaleAfter: Date?` 字段。
- **没引入新依赖**、没动 ABI、没改公共协议签名（仅在 `ProviderSnapshot` 内部加可选参数 + 默认值，对所有调用方向后兼容）。
