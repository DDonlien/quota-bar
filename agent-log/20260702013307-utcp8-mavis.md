## 对话与行动记录

### 调研阶段

**Q1: Antigravity 为什么获取不到订阅额度？**

`md5` 对比结果：
```
===  AntigravityDashboardProvider.swift  main=588 sub=588  md5 一致
===  Strategies.swift                       main=182 sub=182  md5 一致
===  SubscriptionInspector.swift            main=238 sub=238  md5 一致
```

→ Antigravity 的 dashboard provider / pipeline 注册 / inspector 框架在 main 和 sub/main 上**完全一致**。
→ 截图显示 Antigravity 是「待配置 · App 已装」——是 `RefreshCoordinator.applyProviderResult` 在 `AntigravityDashboardProvider` + `KeychainProvider` 全失败后用 `installDetail` fallback 的结果。
→ Antigravity pipeline **没有** BrowserCookie / CLI quota 路径（历史上 REQUIREMENTS v0.2.0-DATA-B-016 只规划了本地 language_server probe），所以「按顺序尝试 CLI 和 Web」在 Antigravity 这个 case 下**不存在可用的 quota fallback**。
→ 要修 Antigravity 必须先抓 Console 日志看 `AntigravityDashboardProvider` 究竟 throw 什么（进程找不到 / 端口找不到 / csrf 失效 / endpoint 协议变了）。

**Q2: Codex 为什么获取不到有效的订阅？**

→ 截图显示「订阅已过期 · plus · 到期 2026/6/25」——这是 v0.8.0 CodexSubscriptionInspector 的**正确输出**。
→ user 反馈昨天 web 续费了，但 `~/.codex/auth.json` 是 Codex.app 在 OAuth flow 时写的本地缓存，web 续费不会触发 Codex.app 重写 auth.json → id_token 里 `chatgpt_subscription_active_until` 仍是 2026/6/25 → inspector 误判过期。
→ `CodexAuthProvider.fetchSnapshot` v0.8.0 设计是 inspector 过期**直接 return marker snapshot**（不 throw），pipeline 立即返回 → 即使 BrowserCookie / CLILog 能拿到真实 quota 也跑不到。
→ `BrowserCookieProvider.fetchSnapshotImpl` Codex 路径 v0.8.0 也加了一次 inspector 调用（"保持一致"），所以**两个 strategy 都被 inspector 短路**了。
→ 即使 pipeline 真的走完，`RefreshCoordinator.applyProviderResult` 在 error 是 `.subscriptionExpired` 但有 `installDetail`（Codex.app 已装）时优先用 `.needsConfiguration(reason: installDetail)`，**又一次**把过期信息覆盖成"待配置"——这是 v0.8.0 设计的隐藏 bug。

### 解决思路

跟 user 对齐：

- **Codex**：v0.8.1 调整 inspector 短路模型。`CodexAuthProvider` 检测到 inspector 过期时**throw** `.subscriptionExpired`，让 pipeline 真走完 BrowserCookie → CLILog → Keychain。`BrowserCookieProvider` Codex 路径移除 v0.8.0 加的 inspector 调用（避免二次短路）。`RefreshCoordinator` 让 `.subscriptionExpired` error 优先用 fallback，不用 installDetail 覆盖。
- **Antigravity**：pipeline 架构上不存在 CLI / Web quota 路径；真正能修的是 DashboardProvider。要先抓 Console 日志定位 throw 原因。这一步**不在本次任务范围**——user 只授权了"do it"（指 Codex），Antigravity 等下次抓日志后再开新任务。

user 选 "do it" → 执行 v0.8.1 修订。
