## 对话与行动记录

### 上一轮总结
v0.5.0 落地后用户验证发现"到期日完全乱写"。根本原因：
- `subscriptionExpiresAt` 的 fallback 逻辑是 `quotas.map(\.resetsAt).max()`
- `resetsAt` 语义是「quota 窗口下次重置时间」不是「订阅到期日」
- 没有任何 provider 显式传 `subscriptionExpiresAt`，所有 provider 走 max 推断

### 关键发现
1. **Kimi 已有真实到期日**：`subscriptionBalance.expireTime` 在 `DashboardEndpoints.swift:378` 已经被解析出来，但被错塞到 Work quota 窗口的 `resetsAt` 字段。**5 行代码就能修好**。
2. **Kimi OAuth 的 `expires_at` 不能用**：那是 access_token 过期时间（小时级），不是订阅到期日。
3. **headless 抓订阅页**比挨个研究私有 API 更通用：订阅管理页就是设计给用户看日期的。
