### 用户诉求

用户要求把 v0.11.0 拆成两个版本：
1. **没有 Apple Developer Program** 的最优雅版本（ad-hoc 预开发版）
2. **有 Apple Developer Program** 的理想实现版本（Developer ID + notarize）

**关键约束**：确保延续性 —— 没有 cert 的情况是**有 cert 情况的前置预开发**，不是两个独立 feature 的平行实现。v0.12.0 应该是 v0.11.0 的**升级**（仅改签名链），而不是 v0.11.0 的**重写**。

### 设计决策

#### 1. Phase 拆分：v0.11.0 = ad-hoc 预开发，v0.12.0 = Developer ID 升级

- **v0.11.0**：
  - 完整跑通 update 机制（UpdateChecker / 状态机 / helper / UI / semver / download）
  - 签名维持 ad-hoc（`--sign -`）
  - helper 脚本也 ad-hoc
  - TCC 权限保留 best-effort（ad-hoc 签名限制）
  - 提前锁定 bundle id `com.taobe.quotabar`（v0.11.0-ARCH-A-000），v0.12.0 升级 cert 时直接复用
- **v0.12.0**：
  - 升级 build script 签名 → Developer ID + `--options runtime`
  - 加 notarize 步骤
  - helper 同步升级
  - cert 一致性断言
  - **不改**：UpdateChecker / 状态机 / UI 框架 / semver workflow

#### 2. 延续性的具体落地

v0.12.0 显式声明"与 v0.11.0 任务的交叉引用"：
