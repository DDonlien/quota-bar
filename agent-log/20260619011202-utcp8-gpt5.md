## 行动记录

- 执行 `git pull`，结果为 `Already up to date`。
- 用本机 `~/.codex/auth.json` 调 `https://chatgpt.com/backend-api/wham/usage`，确认 HTTP 200，返回 `primary_window.used_percent = 30`、`secondary_window.used_percent = 37`。
- 新增 `CodexAuthProvider.swift`，按 CodexBar 的 OAuth 路径优先读取 `~/.codex/auth.json` 并调用 `wham/usage`。
- 将 `ProviderFactory` 改为返回 `PipelineQuotaProvider`，让每个服务按 pipeline 串行 fallback，而不是把所有数据源平铺并发。
- 将 Codex pipeline 调整为 OAuth → Browser Cookie → CLI Log → Keychain。OAuth 成功时不会读取浏览器 Cookie，因此不触发 Full Disk Access。
- 新增 `QuotaFetchError.permissionRequired`，把浏览器 Cookie 权限问题与“未安装/未登录”区分开。
- `RefreshCoordinator` 新增 `needsFullDiskAccess`，只在实际出现浏览器 Cookie 权限问题时驱动 UI 横幅，并过滤 `.notInstalled` snapshot，避免菜单固定展示未安装服务。
- `MenuView` 不再默认调用 `PrivacyAccessChecker.hasFullDiskAccess()`，权限横幅改成按 coordinator 状态显示。
- 修正 `AgentDetector` 的 Codex CLI 登录判断，从错误的 `~/.config/openai` 改为 `$CODEX_HOME/auth.json` 或 `~/.codex/auth.json`。
- 扩展 `CodexDashboardParser`，兼容 `primary_window` / `secondary_window` 与 `five_hour` / `weekly`，并兼容常见 percent 字段名。
- 同步更新 `README.md` 和 `REQUIREMENTS.md`，说明 Codex OAuth 不需要 Full Disk Access，浏览器 Cookie 才按需授权。
