### 3. UI-A-000：验证 expiresAtText==nil 时 HStack 收缩

**review 结果**（基于 v0.3.0-UI-A-010 落地的代码）：

```swift
HStack(spacing: planPriceTrailingGap) {
    if let expiresAtText {
        Text(expiresAtText)...
    }
    Text(snapshot.monthlyPrice ?? "—")...
}
```

- `expiresAtText` 为 nil 时 `if let` 不渲染日期 Text；
- HStack 收缩为只装价格 Text（宽度 ≈ 1 个价格文字）；
- 外层 `Spacer()` 把整个 HStack 推到右边缘。

**结论**：行为已正确，不需要强制改代码。**改了什么**：抽出 `subscriptionExpiresDate: Date?` 公共前置条件（消除 `expiresAtText` 里的内联 guard 重复），加 doc 注释把"nil 时 HStack 收缩"行为锁定在源码里，避免后续维护者误改后产生 layout 跳动。

### 4. UI-A-001：给 expiresAtText 加 tooltip

**改了什么**：

- 新增 `preciseExpiresAtText: String?` 计算属性 + `preciseExpiresAtFormatter`（格式 `yyyy-MM-dd HH:mm:ss zzz`，本地时区，输出形如 `2026-06-25 22:00:00 GMT+8`）。
- 给日期 Text 加 `.help("订阅续费日期：\(preciseExpiresAtText ?? expiresAtText)")`。
- `?? expiresAtText` 兜底：理论上 `preciseExpiresAtText` 和 `expiresAtText` 走同一前置条件都为 nil/同值，加 `??` 只是为 `Optional<String>` 拼接安全，**不会真出现 fallback**。

**为什么用 `zzz`**：`zzz` 输出 `GMT+8` 这种带时区名+偏移的形式，比 `XXXX`（`+0800`）或 `Z`（`+08:00`）更可读；en_US_POSIX locale 保证格式不被本地化。
